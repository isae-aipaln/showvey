# Release Notes v1.1

기간: 2026년 7월 6일

상태: 완료

# **[Showvey v1.1 릴리즈 노트]**

---

### **1. 개요 (Overview)**

품평 운영 중 확인된 데이터 정합성 이슈를 해결하고, 이미지가 없는 스타일도 품평이 가능하도록 평가 플로우를 개선한 업데이트입니다.
또한 개발 도구를 Antigravity에서 **Claude Code**로 전환하고, 커스텀 도메인(**showvey.com**)을 연결했습니다.

---

### **2. 신규 및 개선 기능 (What's New)**

**① 이미지 없는 스타일 품평 지원** — `GalleryPage`, `ProductDetailShell`

- **갤러리**: 상품/코디 이미지가 모두 없는 스타일은 기존 "No Image" 문구 대신 **품번을 카드에 크게 표시**하여 어떤 스타일인지 바로 식별 가능
- **상품 상세**: 이미지가 없는 스타일은 빈 이미지 슬라이더 영역을 숨기고 상품정보/평가 영역을 바로 노출
- **평가 플로우**: 이미지 없는 스타일에서도 평가 입력(가격/구매의사/희망주문량/총평) 후 **"다음 스타일"로 바로 이동** 가능 — 권한별(Staff 1/Staff 2/Store) 평가 항목에 맞춰 저장 처리

**② 갤러리 페이지네이션 개선** — `GalleryPage`

- 상품이 많을 때 페이지 번호가 끝없이 나열되던 문제 개선 → **현재 페이지 중심으로 최대 5개**의 페이지 번호만 표시

**③ 로그인 페이지 저작권 문구 추가** — `LoginPage`

- 로그인 화면 하단에 저작권 고지 추가: `Copyright © 2026 주식회사 이새에프앤씨 All rights reserved.`

### **3. 데이터 정합성 및 버그 수정 (Fixes)**

| 구분 | 내용 | 비고 |
| --- | --- | --- |
| **품번 정규화 강화** | 품번에 포함된 슬래시(`/`)를 언더스코어(`_`)로 자동 변환하여 저장 | Firestore 문서 ID에 `/` 사용 불가 → 품번에 슬래시가 있으면 저장이 깨지던 문제 원천 차단 (`normalizeStyleNo`) |
| **상품 삭제 동기화** | 관리자 품평 편집 화면에서 상품 행을 삭제하고 저장하면 **Firestore의 해당 상품 문서도 함께 삭제** | 기존에는 화면에서만 사라지고 DB에 데이터가 남아 갤러리에 유령 상품이 표시되던 문제 해결 |
| **배포 캐시 버스팅** | 빌드 산출물 파일명에 해시값 포함 (`index-[hash].js`) | 배포 후에도 사용자 브라우저가 구버전 캐시를 계속 보여주던 문제 해결 — 이제 배포 즉시 새 버전 반영 |

### **4. 인프라 및 운영 (Infra & Ops)**

- **커스텀 도메인 연결**: 서비스 주소를 **https://showvey.com** 으로 변경 (기존 https://showvey.web.app 도 유효)
- **개발 도구 전환**: Antigravity 사용량 초과에 따라 **Claude Code** 기반의 수정 → 빌드 → 검토(로컬 미리보기) → 배포 워크플로우로 전환 완료
- **GitHub 버전 관리**: 금번 변경사항 전체를 원격 저장소에 커밋/푸시하여 **GitHub = 로컬 소스 = 실서버** 삼자 동기화 (커밋 `862b5b3`)

### **5. 기술 스택 (Tech Stack)**

- **Frontend**: React, TypeScript, Vite, Tailwind CSS, shadcn-ui
- **Backend / DB**: Firebase Authentication, Cloud Firestore, Cloud Storage
- **Hosting**: Firebase Hosting + 커스텀 도메인 (showvey.com)
